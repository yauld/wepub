# 产品截图指南

好的截图能让第一次打开仓库的人快速理解 wepub 解决了什么问题。

## 推荐截图

| 优先级 | 截图 | 建议文件名 | 说明 |
| --- | --- | --- | --- |
| P0 | 主工作台 | `docs/images/wepub-workspace.png` | 左侧文档源、右侧预览、顶部按钮都应可见 |
| P0 | 同步到草稿箱弹窗 | `docs/images/wepub-draft-sync.png` | 展示标题、摘要、封面和确认按钮 |
| P0 | 微信公众号草稿箱或编辑器结果 | `docs/images/wepub-wechat-draft.png` | 展示草稿已创建，重点看代码块、图片、列表是否正常 |
| P1 | 文档目录 | `docs/images/wepub-local-assets.png` | 展示 `.md` / `.ipynb` 与 `assets/` 目录放在一起 |

## 截图要求

- 使用浅色模式，宽度建议 1600px 以上。
- 遮住 AppID、AppSecret、access_token、真实公网 IP、账号主体信息等敏感内容。
- 如果公众号名称不想公开，也请遮住。
- 尽量使用真实技术文章，但避免露出未发布的商业信息。
- 截图文件建议控制在 1 MB 内。

## README 使用策略

- README 顶部只放一张主工作台截图，避免首屏过重。
- “同步到微信公众号草稿箱”章节可以补 1-2 张流程截图。
- 公众号后台成功截图最能建立信任，但必须先确认没有敏感信息。
